# B001 화면 제작 프롬프트 (교정본)

## 1. 개요

회사 소스 구조를 토대로 개발 환경을 구성하고, 아래 요구사항에 따라 백엔드·프론트엔드를 구현하여 학습한다.

### 백엔드 (Spring Boot)

- API를 통해 서비스에서 DB 연결 없이 미리 만들어 둔 리스트를 반환한다.

### 프론트엔드 (React)

- AG Grid를 이용해 목록을 표시한다.
- 화면에 조회 조건은 **6개**가 있어야 한다.

---

## 2. 그리드 조회 동작

- 화면 로드 직후에는 그리드 목록을 **조회하지 않는다**.
- **조회 버튼**을 클릭했을 때만 그리드 데이터를 조회한다.

---

## 3. 필터 항목

### 1번 필터: 주차 달력 (DatePicker)

- 화면 로드 직후 **오늘 날짜의 주차**가 자동 선택되어야 한다.
- 달력에서 `2026년 6월 29일`을 선택하면 값은 `"202627"`이며, 선택한 값을 필터에 표시해야 한다.

### 2번 필터: 셀렉트박스

- 1번 필터의 값으로 API를 조회한 뒤, 2번 셀렉트박스에 목록을 표시한다.
- 목록 중 **첫 번째 항목**을 자동 선택한다.

### 3번 필터: 셀렉트박스

- 1번 필터의 값과 2번 필터의 **코드**로 API를 조회한 뒤, 3번 셀렉트박스에 목록을 표시한다.
- 목록의 **첫 번째 항목은 "전체"**여야 하며, 첫 번째 항목을 자동 선택한다.

### 4번 필터: 셀렉트박스

- API로 조회한 리스트를 셀렉트박스에 표시한다.
- **다중 선택**이 가능해야 한다.

### 5번 필터: 라디오박스

- 라디오박스 **2개**.

### 6번 필터: 체크박스

- 체크박스 **4개**.

---

## 4. 공통 사항

- 모든 셀렉트박스, 라디오, 체크박스는 **코드(code)** 와 **이름(name)** 으로 구성한다.
- 선택 후 조회 시 **코드**가 파라미터로 API 조회 조건에 전달되어야 한다.

---

## 5. 리셋 버튼

- 조회 버튼 옆에 **리셋** 버튼이 있어야 한다.
- 리셋 버튼 클릭 시 조회 조건 값이 초기화되어야 한다.

---

## 6. API 호출 예시

```javascript
export const fetchGridDepartments = async (params) => {
  let searchData = params;
  const response = await axiosUtil.get("api/grid/options/departments", searchData, {});
  return response;
};
```

## 7. 패키지 기본 구조

```
api
  └ B001Api.js
components
  └ SearchPanel.jsx
  └ GridPanel.jsx
hooks
  └ useSearch.js
  └ useGrid.js
pages
  └ B001Page.jsx
  └ StudyPage.jsx
```

- `SearchPanel`: 조회 항목(필터) 모음
- `GridPanel`: AG Grid 사용
- `B001Page`: `SearchPanel`, `GridPanel` 컴포넌트를 각각 사용
